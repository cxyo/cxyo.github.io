# 基金温度表多平台部署详细操作文档

## 一、概述

本文档详细介绍基金温度表应用在各个主流平台的部署方法，从环境准备到最终上线，每一步都配有详细说明。无论您是初学者还是有经验的开发者，都可以根据自己的需求选择合适的平台完成部署。基金温度表是一个纯前端应用，不需要后端服务器支持，这使得它可以部署在任何静态文件托管平台上，大大降低了部署难度和成本。

本应用的技术特点决定了部署的灵活性：它只依赖HTML、CSS、JavaScript和CSV/JSON数据文件，不涉及数据库、动态服务器端渲染或任何需要环境变量的配置。因此，我们可以选择从最简单的本地预览，到功能完善的云端部署等多种方案。不同方案各有优劣，本文档将详细介绍每种方案的适用场景和操作步骤，帮助您找到最适合自己的部署方式。

---

## 二、本地部署

本地部署是最基础的部署方式，适合在开发阶段进行测试和调试，或者在没有网络的环境下使用。部署过程不需要任何账户注册，完全免费且速度快，是每位开发者必须掌握的基础技能。

### 2.1 使用VS Code Live Server插件部署（推荐）

Visual Studio Code（简称VS Code）是目前最流行的代码编辑器之一，它提供了丰富的扩展生态，Live Server扩展是其中最受欢迎的Web开发工具之一。使用Live Server部署的步骤如下：

**第一步：下载安装VS Code编辑器**

访问VS Code官方网站（https://code.visualstudio.com/），点击首页的下载按钮，根据您的操作系统（Windows、macOS或Linux）选择对应的安装包。下载完成后，运行安装程序，按照向导提示完成安装。安装过程非常简单，只需要点击"下一步"同意许可协议，选择安装位置即可。安装完成后，启动VS Code，您会看到一个简洁的界面，左侧是文件浏览器，中间是代码编辑区，底部是终端。

**第二步：安装Live Server扩展**

在VS Code界面左侧找到扩展图标（或者按快捷键Ctrl+Shift+X），点击打开扩展面板。在搜索框中输入"Live Server"，找到由Ritwick Dey开发的Live Server扩展，点击"安装"按钮。安装完成后，您可能需要重启VS Code使扩展生效。Live Server扩展会自动监测文件变化并在保存时自动刷新浏览器，大大提高了开发效率。

**第三步：打开项目文件夹**

在VS Code中点击"文件"菜单，选择"打开文件夹"，导航到fund-main文件夹并选中，点击"选择文件夹"。左侧文件浏览器中会显示项目的所有文件结构，包括index.html、css/style.css、js/app.js、code.json、日期命名的CSV文件（如2026-01-06.csv）以及css、js、images等子文件夹。您可以点击各个文件查看其内容。

**第四步：启动本地服务器**

在fund-main文件夹中找到index.html文件，右键点击它，在右键菜单中选择"Open with Live Server"。或者，您也可以在index.html文件打开的状态下，在编辑器右下角找到"Go Live"按钮并点击。Live Server会自动启动默认浏览器，访问http://127.0.0.1:5500/，您将看到基金温度表应用正在运行。此时，任何对代码的修改都会自动反映到浏览器中，无需手动刷新。

### 2.2 使用Python简单HTTP服务器部署

如果您已经安装了Python环境，可以使用Python内置的HTTP模块快速启动一个本地服务器。这种方式不需要安装任何额外软件，适合临时测试或没有安装VS Code的情况。

**第一步：打开命令行工具**

Windows用户按Win+R，输入cmd回车；或者右键点击开始菜单，选择"Windows PowerShell"。macOS用户打开Terminal应用，Linux用户打开终端。

**第二步：导航到项目目录**

使用cd命令导航到fund-main文件夹，例如：`cd d:\AI_CODE_project\DEMO\fund-main`。如果路径中包含空格或特殊字符，建议使用引号将路径括起来：`cd "d:\AI_CODE project\DEMO\fund-main"`。

**第三步：启动HTTP服务器**

确认路径正确后，执行启动服务器命令。对于Python 3版本，执行：`python -m http.server 8000`。这条命令的意思是使用Python的http模块，以8000端口启动一个简单的Web服务器。服务器启动后，您会看到类似以下的输出：
```
Serving HTTP on ::8000 ...
```
这表示服务器已经在8000端口监听请求。

**第四步：访问应用**

服务器启动后，打开浏览器访问http://localhost:8000/，您将看到基金温度表应用正在运行。服务器会在当前目录及其子目录中查找文件，index.html会被自动识别为默认首页。

**其他说明**

要停止服务器，在命令行窗口中按Ctrl+C即可。如果8000端口已被占用，您可以更换端口号，例如使用8080端口：`python -m http.server 8080`，然后访问http://localhost:8080/。

### 2.3 使用Node.js http-server部署

Node.js是一个基于Chrome V8引擎的JavaScript运行环境，它提供了丰富的包管理工具。对于前端开发者来说，Node.js几乎是必备工具。如果您已经安装了Node.js，可以使用http-server包来启动本地服务器。

**第一步：确认Node.js已安装**

打开命令行，执行`node --version`，如果显示版本号（如v18.16.0），说明Node.js已安装。如果没有安装，请访问Node.js官网（https://nodejs.org/），下载LTS（长期支持）版本并安装。安装过程会自动添加环境变量，安装完成后需要重启命令行工具。

**第二步：安装http-server**

在命令行中执行`npm install -g http-server`，全局安装http-server工具。安装过程可能需要一些时间，耐心等待安装完成。

**第三步：启动服务器**

导航到fund-main文件夹，执行`http-server -p 8080`，服务器将在8080端口启动。您会看到服务器输出访问地址，例如：
```
Starting up http-server, serving ./
Available on:
  http://127.0.0.1:8080
  http://192.168.1.100:8080
```

**第四步：访问应用**

打开浏览器访问http://127.0.0.1:8080，即可查看应用。http-server支持很多参数，例如`-c-1`禁用缓存（方便开发调试）、`--ssl`启用HTTPS等，您可以根据需要添加参数。

## 三、GitHub Pages部署

GitHub Pages是GitHub提供的静态网站托管服务，完全免费且与GitHub仓库紧密集成。它支持自定义域名，启用HTTPS，非常适合托管开源项目的文档、演示页面或个人博客。部署后，您的网站将可以通过类似https://用户名.github.io/仓库名/的地址访问。

### 3.1 创建GitHub账户

**第一步：访问GitHub官网**

打开浏览器访问https://github.com/，您会看到GitHub的首页，上面展示了各种开源项目和功能介绍。

**第二步：开始注册**

点击页面右上角的"Sign up"按钮开始注册。在注册页面，您需要依次完成以下步骤：

填写电子邮箱地址：建议使用常用的邮箱，因为后续找回密码、接收通知等都会用到。输入邮箱后，点击"Continue"。

创建账户：设置用户名（Username），用户名将成为您仓库地址的一部分，建议使用简洁且容易记忆的名称，例如"mygithub"。用户名只能包含字母、数字和连字符，且不能以连字符开头。设置密码，密码至少需要8个字符，包含字母和数字。完成后点击"Continue"。

验证身份：GitHub会要求您完成一个简单的拼图验证，证明您不是机器人。根据页面提示完成验证即可，可能需要拖动滑块或点击特定图片。

选择计划：GitHub提供免费和付费两种计划，选择免费的"Free"计划即可满足我们的需求。点击"Continue"完成注册。

**第三步：验证邮箱**

注册完成后，GitHub会向您的邮箱发送一封验证邮件。打开邮件，点击验证链接完成邮箱验证。如果您在收件箱中没有看到邮件，请检查垃圾邮件文件夹。验证成功后，您就可以正常使用GitHub的所有功能了。

### 3.2 创建代码仓库

仓库（Repository）是存储代码的地方，每个项目通常对应一个独立的仓库。创建仓库的步骤如下：

**第一步：进入创建页面**

登录GitHub后，点击页面右上角的"+"号，选择"New repository"。或者直接访问https://github.com/new进入创建页面。

**第二步：填写仓库信息**

在创建页面，填写以下信息：

Repository name：仓库名称，建议使用英文，例如"fund-temperature-table"。这个名称会成为URL的一部分，只能包含字母、数字、连字符和下划线。

Description（可选）：仓库描述，例如"基金温度表应用 - 指数估值可视化工具"。这会帮助其他人了解您的项目。

选择仓库可见性：选择"Public"（公开），这样部署后任何人都可以访问。如果您选择Private，GitHub Pages将无法公开访问。

初始化选项：不勾选"Add a README file"（因为我们已经有完整的项目文件）；不添加.gitignore和license（这些我们已经在项目中准备好）。

**第三步：创建仓库**

填写完成后，点击"Create repository"按钮，仓库就创建好了。您会看到一个空仓库的页面，提示您可以上传文件或创建新文件。

### 3.3 上传代码到仓库

代码上传有多种方式，我们介绍最常用的Git命令行方式和网页上传方式。

**方式一：使用Git命令行上传**

首先，下载并安装Git。访问Git官网（https://git-scm.com/），下载适合您操作系统的版本。安装过程保持默认设置即可，安装完成后在命令行中执行`git --version`验证安装。

在fund-main文件夹中右键打开Git Bash（或在命令行中导航到fund-main文件夹）。初始化Git仓库，执行`git init`，这会在文件夹中创建一个隐藏的.git目录用于版本控制。

配置用户信息（如果尚未配置），执行以下命令：
```
git config --global user.name "您的GitHub用户名"
git config --global user.email "您的GitHub注册邮箱"
```

将所有文件添加到暂存区，执行`git add .`。这个命令会将文件夹中所有文件（包括子文件夹中的文件）添加到Git的暂存区，准备提交。

提交代码，执行`git commit -m "Initial commit"`。双引号中的内容是提交说明，建议写清楚本次提交的内容。执行后，Git会显示提交的文件数量和变更统计。

关联远程仓库。执行以下命令（将"用户名"替换为您的GitHub用户名）：
```
git remote add origin https://github.com/用户名/fund-temperature-table.git
```

推送代码到GitHub，执行`git push -u origin master`（或main，取决于Git默认分支名称）。第一次推送需要输入GitHub用户名和密码（或Personal Access Token）进行身份验证。代码上传成功后，刷新GitHub仓库页面，您会看到所有文件已经上传成功，包括index.html、css文件夹、js文件夹、images文件夹等。

**方式二：使用网页上传**

如果您不熟悉Git命令，也可以通过网页直接上传文件。在GitHub仓库页面，点击"uploading an existing file"链接进入上传页面。将fund-main文件夹中的所有文件拖拽到网页上（注意：需要上传文件夹内的所有内容，包括css、js、images等子文件夹，不能只上传文件夹本身）。填写提交说明（Commit changes），点击"Commit changes"完成上传。

### 3.4 启用GitHub Pages

代码上传后，启用GitHub Pages进行部署。

**第一步：进入设置页面**

在GitHub仓库页面，点击仓库名称下方的"Settings"选项卡进入设置页面。

**第二步：找到Pages设置**

在左侧菜单中找到"Pages"选项，点击进入GitHub Pages设置页面。

**第三步：配置部署选项**

在"Build and deployment"部分，确保Source设置为"Deploy from a branch"。在"Branch"部分，选择分支（通常是master或main），文件夹选择"/ (root)"（即根目录），然后点击"Save"。

**第四步：获取访问地址**

等待一两分钟，页面会刷新并显示您的网站链接，格式为https://用户名.github.io/仓库名/。点击链接即可访问部署好的应用。如果链接没有立即出现，请稍等片刻后刷新页面。

### 3.5 自定义域名（可选）

如果您有自己的域名，可以将其绑定到GitHub Pages。

**第一步：添加自定义域名**

在GitHub Pages设置页面，找到"Custom domain"输入框，输入您的域名（如fundtemp.example.com），点击"Save"。

**第二步：配置DNS记录**

在您的域名服务商处添加CNAME记录，将域名指向您的GitHub Pages地址（用户名.github.io）。具体的配置方法因域名服务商而异，一般需要添加一条CNAME记录，主机记录为"@"（表示主域名）或您要使用的子域名，记录值为您的GitHub Pages地址。

**第三步：启用HTTPS**

保存域名设置后，GitHub会自动为您的域名申请和配置SSL证书（可能需要几分钟到几小时）。配置完成后，勾选"Enforce HTTPS"选项，强制使用HTTPS访问。

### 3.6 使用GitHub Actions自动更新数据

本项目包含一个GitHub Actions工作流，可以自动更新基金净值数据和基金温度数据，避免频繁调用API导致被封号，同时提高前端页面加载速度。

**第一步：了解工作流功能**

- 工作流名称：Update Fund Data
- 触发方式：定时触发（每天北京时间晚上8点）和手动触发
- 功能：
  - 爬取所有金融数据（包括国债收益率、A股数据、GDP数据等）
  - 预处理所有CSV文件，计算每天每个基金的温度数据
- 输出：
  - 生成 `data/fund-nav-data.json` 文件（基金净值数据）
  - 生成 `fund_temperature.json` 文件（基金温度数据）
  - 生成 `combined_data.json` 文件（合并所有金融数据）
- 好处：
  - 避免页面访问时频繁调用API，提高页面加载速度
  - 前端直接加载JSON数据，不再需要处理CSV文件
  - 基金温度计算更加准确，支持显示所有历史数据

**第二步：手动触发工作流**

1. 登录GitHub账号，进入仓库页面
2. 点击顶部导航栏的「Actions」选项卡
3. 在左侧找到「Update Fund Data」工作流
4. 点击「Run workflow」按钮
5. 点击「Run workflow」按钮开始更新数据

**第三步：处理403权限错误**

如果您在运行工作流时遇到类似以下错误：
```
remote: Permission to cxyo/fund.git denied to github-actions[bot].
fatal: unable to access 'https://github.com/cxyo/fund/': The requested URL returned error: 403
```

这是因为GitHub Actions机器人没有足够的权限推送更改到仓库。解决方法如下：

**方法1：修改仓库权限设置**

1. 进入仓库页面，点击「Settings」→「Actions」→「General」
2. 找到「Workflow permissions」部分
3. 选择「Read and write permissions」
4. 点击「Save」保存设置

**方法2：创建Personal Access Token (PAT)**

如果方法1不起作用，您可以创建一个Personal Access Token来授予GitHub Actions推送权限：

1. 点击GitHub头像 →「Settings」（设置）→「Developer settings」（开发者设置）→「Personal access tokens」（个人访问令牌）→「Tokens (classic)」（经典令牌）
2. 点击「Generate new token」（生成新令牌）→「Generate new token (classic)」（生成新的经典令牌）
3. 在「Note」（说明）字段填写一个描述，例如「GitHub Actions Fund Data Update」
4. 在「Select scopes」（选择作用域）部分，勾选「repo」权限组下的所有选项
5. 滚动到页面底部，点击「Generate token」（生成令牌）
6. 复制生成的Token，妥善保存（注意：此Token只会显示一次，刷新页面后将无法再次查看）
7. 进入仓库页面，点击「Settings」（设置）→「Secrets and variables」（密码和变量）→「Actions」（Actions）
8. 点击「New repository secret」（新建仓库密码）
9. 在「Name」（名称）字段填写「GH_PAT」，在「Secret」（密码）字段粘贴刚才复制的Token
10. 点击「Add secret」（添加密码）
11. 修改`.github/workflows/update-fund-data.yml`文件，将最后一步的git push命令改为使用Token：
    ```yaml
    - name: Commit and push changes
      run: |
        git config --global user.name 'GitHub Actions'
        git config --global user.email 'actions@github.com'
        git add bond_yield.txt bond_yield.json a_stock_data.json gdp_data.json combined_data.json fund_temperature.json
        git commit -m 'Update financial data'
        git push https://${{ secrets.GH_PAT }}@github.com/${{ github.repository }}.git
    ```

**注意**：当前默认的工作流文件已经使用了直接的git push命令，如果您遇到403权限错误，可以尝试方法1，或者按照上述步骤修改工作流文件使用Token认证。

**第四步：使用生成的数据文件**

工作流成功运行后，会在仓库中生成以下文件：

1. `data/fund-nav-data.json`：基金净值数据，页面刷新时会直接读取该JSON文件，不再调用API
2. `fund_temperature.json`：基金温度数据，前端通过Ajax加载该文件，大幅提高图表加载速度
3. `combined_data.json`：合并所有金融数据，减少前端请求次数

**第五步：基金温度数据更新验证**

工作流成功运行后，您可以通过以下方式验证基金温度数据是否正确更新：

1. 查看仓库中的 `fund_temperature.json` 文件是否存在且包含数据
2. 本地运行 `python generate_temperature_json.py` 命令，检查输出是否正常
3. 访问部署的页面，点击任意基金行，查看历史温度图表是否加载迅速且数据正确

通过GitHub Actions自动更新数据，您可以确保基金温度表始终使用最新的数据，同时享受更快的页面加载速度。

## 四、Netlify部署

Netlify是一家提供现代Web项目托管服务的公司，专注于静态网站和前端应用。它提供了慷慨的免费套餐，包含无限流量、绑定自定义域名、免费SSL证书等功能。Netlify的最大特点是支持拖拽部署——您只需要将项目文件夹拖到浏览器中，就能完成部署。

### 4.1 注册Netlify账户

**第一步：访问Netlify官网**

打开浏览器访问https://www.netlify.com/，您会看到Netlify的首页，展示了各种功能特点。

**第二步：开始注册**

点击首页的"Sign up"按钮开始注册。Netlify支持多种注册方式：

Email邮箱注册：点击"Sign up with email"，填写邮箱地址和密码，完成验证后即可注册。

GitHub账户授权（推荐）：点击"Sign up with GitHub"，在弹出的授权页面点击"Authorize Netlify"完成授权。这样稍后部署GitHub仓库时会更加方便。

GitLab账户授权或Bitbucket账户授权：如果您使用其他代码托管平台，可以选择相应的授权方式。

**第三步：进入控制台**

授权或注册成功后，您会进入Netlify的控制台（Dashboard）界面，这是管理所有站点的中心。左侧菜单可以查看所有站点、部署记录、团队成员等信息。

### 4.2 部署方式一：拖拽部署（最简单）

拖拽部署是Netlify最便捷的部署方式，不需要任何命令行操作。

**第一步：准备部署文件**

在本地将fund-main文件夹压缩成zip文件（注意：压缩包内应直接包含所有文件，而不是嵌套一层文件夹）。右键点击fund-main文件夹，选择"发送到"->"压缩(zipped)文件夹"，或者使用压缩软件创建zip文件。

**第二步：拖拽上传**

打开Netlify控制台，将下载的zip文件拖拽到页面中央的虚线框内。Netlify会自动解压并开始部署站点，页面上会显示部署进度。

**第三步：访问应用**

部署完成后，页面会显示您的临时站点地址（格式类似于random-name-12345.netlify.app），点击即可访问。您可以在控制台中点击"Site settings"进行进一步配置，如重命名站点、绑定自定义域名等。

拖拽部署适合快速测试或一次性部署，如果您需要持续更新代码，建议使用GitHub仓库集成部署方式。

### 4.3 部署方式二：GitHub仓库集成部署

这种方式可以让您每次更新GitHub仓库时自动触发Netlify重新部署，实现持续部署。

**第一步：连接GitHub**

在Netlify控制台点击"Add new site"，选择"Import an existing project"。第一次使用时需要点击"Configure Netlify on GitHub"进行权限授权，选择要授权的仓库（可以授权所有仓库或仅特定仓库）。

**第二步：选择仓库**

授权完成后，从下拉列表中选择您之前创建的fund-temperature-table仓库。

**第三步：配置部署选项**

在部署设置页面，Netlify会自动识别这是一个纯前端项目。

Branch选择main（或master）。

Build command留空（因为我们是纯静态网站，不需要构建）。

Publish directory保持默认的根目录"/"。

点击"Deploy site"开始部署。Netlify会自动拉取代码、部署站点。

**第四步：配置持续部署**

部署完成后，每次您向GitHub仓库推送代码，Netlify都会自动重新部署站点。您可以在Netlify控制台的"Deploys"标签页查看部署历史和状态。如果部署失败，Netlify会发送邮件通知您。

### 4.4 绑定自定义域名

**第一步：进入域名管理**

在Netlify控制台进入您的站点，点击"Domain management"选项卡。

**第二步：添加自定义域名**

在"Custom domains"部分点击"Add custom domain"，输入您的域名并点击"Verify"。

**第三步：配置DNS记录**

选择域名类型（通常选择"Primary domain"），然后按照页面提示在域名服务商处配置DNS记录。

对于主域名（如example.com），通常添加A记录指向Netlify的负载均衡IP地址（Netlify会在页面上显示具体IP地址）。

对于子域名（如www.example.com），建议添加CNAME记录指向您的Netlify站点地址（xxx.netlify.app）。

**第四步：等待生效**

配置完成后，等待DNS生效（可能需要几分钟到几小时）。DNS生效后，您的域名就可以正常访问了。Netlify会自动为您的域名配置SSL证书。

## 五、Vercel部署

Vercel是另一个优秀的静态网站托管平台，与Next.js框架同属一家公司。Vercel对GitHub集成支持非常好，部署速度极快，同样提供免费套餐和自定义域名支持。如果您使用Vercel部署，通常可以在仓库更新后几十秒内看到新版本上线。

### 5.1 注册Vercel账户

**第一步：访问Vercel官网**

打开浏览器访问https://vercel.com/，您会看到Vercel的首页。

**第二步：开始注册**

点击"Sign up"开始注册。Vercel同样支持多种注册方式，推荐使用GitHub账户授权注册，因为这样可以直接导入GitHub仓库进行部署。

**第三步：GitHub授权**

点击"Continue with GitHub"，在弹出的授权页面点击"Authorize vercel"完成授权。如果弹出窗口被拦截，请允许弹出窗口并重试。

**第四步：进入控制台**

授权成功后，您会进入Vercel的控制台界面。界面简洁明了，可以查看所有项目、部署记录等信息。

### 5.2 导入GitHub仓库部署

**第一步：创建新项目**

在Vercel控制台点击"Add New..."，选择"Project"。

**第二步：连接GitHub**

第一次使用时需要点击"Import GitHub Repository"连接到您的GitHub账户，选择需要部署的仓库（fund-temperature-table）。如果仓库没有出现在列表中，点击"Configure GitHub App"检查授权设置。

**第三步：配置项目**

在项目配置页面，Vercel会自动识别这是一个纯前端项目。Framework Preset保持默认即可，Root Directory保持默认。Build Command和Output Directory都留空（静态网站不需要构建）。

**第四步：部署**

点击"Deploy"开始部署。Vercel会自动拉取代码、部署站点。部署过程中，页面会显示构建日志和进度。部署完成后，您会看到访问链接（格式为https://项目名.vercel.app），点击即可访问。

### 5.3 持续部署配置

Vercel会自动配置持续部署。每次您向GitHub仓库推送代码，Vercel都会自动检测并触发新部署。您可以在Vercel控制台的"Deployments"标签页查看所有部署记录和状态，包括部署时间、部署状态（就绪/失败）、构建日志等信息。

要修改部署配置或绑定自定义域名，进入站点详情页，点击"Settings"->"Domains"，添加您的自定义域名即可。Vercel同样会自动为自定义域名配置SSL证书。

## 六、腾讯云部署

腾讯云是国内领先的云服务提供商，提供了多种静态网站托管方式。对于面向国内用户的应用，使用国内云服务商可以确保访问速度和稳定性。

### 6.1 注册腾讯云账户

**第一步：访问腾讯云官网**

打开浏览器访问https://cloud.tencent.com/，您会看到腾讯云首页。

**第二步：注册账户**

点击右上角的"免费注册"或"立即注册"。支持微信扫码、手机号、邮箱等多种注册方式。按照提示完成注册流程。

**第三步：实名认证**

使用云服务需要完成实名认证。登录腾讯云控制台后，按照页面提示上传身份证信息或企业资质进行认证。实名认证通常需要1-3个工作日完成。

### 6.2 腾讯云对象存储静态网站托管（推荐）

对于前端应用，腾讯云的对象存储（COS）配合静态网站托管是很好的选择。

**第一步：创建存储桶**

在腾讯云控制台顶部搜索框输入"对象存储"，在搜索结果中选择"对象存储 COS"进入COS服务页面。点击"创建存储桶"，填写以下信息：

存储桶名称：例如"fund-temp-bucket"（需要全局唯一）。

地域：选择离您的主要用户最近的地域，如"广州"或"上海"。

访问权限：选择"公有读私有写"，这样所有人才能访问您的网站。

其他设置保持默认即可。填写完成后点击"创建"。

**第二步：上传文件**

进入存储桶管理页面，点击"上传文件"，将fund-main文件夹内的所有文件上传到存储桶根目录。注意：需要上传所有文件，包括css、js、images子文件夹中的文件，以及日期命名的CSV文件。

**第三步：开启静态网站托管**

在存储桶设置中找到"静态网站托管"选项，点击"设置"。点击"开启静态网站托管"，指定索引文档为"index.html"，错误文档也为"index.html"。点击"保存"。

**第四步：获取访问地址**

保存设置后，页面会显示您的静态网站访问地址，格式为https://存储桶名-账户ID.cos.website.地域.myqcloud.com。点击链接即可访问。

### 6.3 绑定自定义域名

**第一步：进入域名管理**

在存储桶的"域名管理"页面，点击"自定义域名"。

**第二步：添加域名**

点击"添加域名"，输入您的自定义域名（如fundtemp.example.com）。

**第三步：配置DNS记录**

根据页面提示，在您的域名服务商处配置DNS记录。通常需要添加一条CNAME记录，记录值指向腾讯云提供的加速域名。

**第四步：等待生效**

DNS配置完成后，等待几分钟即可通过自定义域名访问您的网站。腾讯云会自动为您的域名配置SSL证书。

## 七、阿里云部署

阿里云是国内另一家主要的云服务提供商，其对象存储OSS服务同样支持静态网站托管。

### 7.1 注册阿里云账户

访问阿里云官网（https://www.aliyun.com/），完成注册和实名认证。流程与腾讯云类似，这里不再赘述。

### 7.2 阿里云对象存储OSS部署

**第一步：创建Bucket**

登录阿里云控制台，搜索"对象存储 OSS"，进入OSS服务。点击"创建Bucket"，填写存储桶名称、地域，选择"标准存储"类型，访问权限选择"公共读"，其他设置保持默认。

**第二步：上传文件**

进入Bucket，点击"文件管理"，上传fund-main文件夹内的所有文件到根目录，包括日期命名的CSV文件。

**第三步：开启静态网站托管**

在Bucket的"基础设置"中找到"静态网站托管"选项，点击"设置"。开启静态网站托管，设置默认首页为index.html，默认404页为index.html。

**第四步：访问应用**

保存设置后，获取Bucket的访问域名，通过该域名即可访问您的应用。

## 十、常见问题与解决方案

### 10.1 页面显示空白或数据加载失败

检查浏览器控制台（按F12打开开发者工具，切换到Console标签），查看错误信息。

如果报错"Access to fetch at 'file://...' from origin 'null' has been blocked by CORS policy"，说明您正在直接打开本地HTML文件。解决方法是使用本地服务器（如Live Server）访问。

如果报错找不到data.csv或code.json文件，检查这些文件是否在正确的位置。

### Q5: CSV数据加载失败？
A: 确保所有日期命名的CSV文件（如2026-01-06.csv）位于仓库根目录，且文件名正确。系统会自动扫描本地目录下的所有CSV文件，按文件名日期降序排序，自动加载最新的两个文件。

**智能加载机制：**
- 完全基于本地CSV文件的实际名称，不依赖今天明天的日期
- 自动识别最新的两个CSV文件，无需手动指定日期
- 支持文件不连续的情况（如缺少1、2、3、4、10、11等日期）
- 每次增加或减少CSV文件，应用都会自动更新数据，无需修改任何代码
- 上一日涨跌使用最新CSV文件的涨跌幅
- 上两日涨跌使用次新CSV文件的涨跌幅

GitHub Pages对大小写敏感。系统会自动加载所有日期命名的CSV文件。

### Q6: 如何查看基金温度历史图表？
A: 点击基金温度表中的任意指数行，即可显示该指数的基金温度历史记录折线图。图表功能包括：
- 自动扫描文件夹内所有CSV文件
- 计算每个日期的温度值
- 支持选择时间范围（5天、10天、15天、20天）
- 显示最高和最低温度标记（最高温度红色，最低温度绿色）
- 平滑曲线和渐变填充效果
- 鼠标悬停显示详细信息

**温度计算规则：**
- 行业指数（E类）：温度 = PB分位点 × 100
- 其他指数：温度 = (PE分位点 + PB分位点) ÷ 2 × 100

### 10.2 数据不更新

部署新版本后，尝试强制刷新浏览器（按Ctrl+F5），或在浏览器设置中清除缓存。

对于Netlify/Vercel部署，检查部署日志看是否有错误。

### 10.3 自定义域名无法访问

检查DNS配置是否正确，使用在线DNS查询工具检查域名解析。

确保域名已正确添加SSL证书。

### 10.4 页面样式丢失

检查CSS文件是否正确加载，在浏览器开发者工具的Network标签中查看css/style.css状态。
